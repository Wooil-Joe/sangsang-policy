🚀 앱 진입점 & 초기화 Core
▼
lib/main.dart
앱의 시작점. Firebase 초기화, 테마 설정, 라우팅 연결을 담당합니다. kUseDataConnectEmulator 플래그로 에뮬레이터/프로덕션 전환.
→ 만들어내는 것: 앱 부팅 → Firebase 연결 → 첫 화면 표시
lib/routes/
화면 간 네비게이션 규칙. Named route 방식으로 /main, /login, /post-detail 등의 경로를 정의합니다.
→ 만들어내는 것: 사용자가 버튼 누르면 해당 화면으로 이동하는 흐름
🔐 인증 & 회원가입 Service
▼
lib/services/auth_service.dart
로그인/로그아웃/회원탈퇴 핵심 로직. Google, Apple, Kakao 3가지 소셜 로그인을 처리합니다. Kakao는 Firebase Custom Token 방식으로 연동 (Cloud Functions 경유).
→ 만들어내는 것: 소셜 로그인 버튼 클릭 → Firebase 인증 → 앱 진입
lib/screens/sign_up_screen.dart
회원가입 멀티스텝 UI. PageView 기반 5스텝(디자이너), 5스텝(셀러), 3스텝(일반). 약관 동의, Pro/Indie 선택, 사업자 번호 검증(국세청 API), 프로필 입력까지.
→ 만들어내는 것: 단계별 가입 화면 + 유저 타입별 분기 + DB 저장
lib/screens/login_screen.dart
로그인 화면. 이메일/비밀번호 + 소셜 로그인 버튼 + "기억하기" (flutter_secure_storage AES-256 암호화).
→ 만들어내는 것: 로그인 UI + 자동 로그인 + 비밀번호 복호화

💾 데이터 레이어 (DB 통신) Service
▼
lib/services/dataconnect_service.dart
가장 큰 서비스 파일. 모든 DB 읽기/쓰기의 허브. Firebase Data Connect SDK를 통해 GraphQL 쿼리/뮤테이션을 실행합니다. 게시물 CRUD, 협업 생성/상태 변경, 견적서, 정산 등 모든 데이터 조작이 여기를 거칩니다.
→ 만들어내는 것: "이 함수를 호출하면 → DB에 데이터가 저장/조회된다"의 중심
dataconnect/schema/schema.gql
DB 테이블 설계도 (v6.3). User, Post, Cowork, Quotation, Payment, SampleProgress, Production, CSRequest, Settlement 등 모든 테이블의 컬럼과 관계를 정의합니다. 이 파일이 바뀌면 → SDK 재생성 필요.
→ 만들어내는 것: PostgreSQL에 실제 테이블이 생성됨 (마이그레이션)
dataconnect/sangsang/queries.gql
데이터 읽기 정의. GetPostById, ListPosts, GetCoworkDetail, GetMyPayments 등 "어떤 데이터를 어떻게 조회할지" 정의합니다.
→ 만들어내는 것: "홈 피드에 보여줄 게시물 목록" 같은 쿼리 결과
dataconnect/sangsang/mutations.gql
데이터 쓰기 정의. CreatePost, CreateCowork, UpdateCoworkStatus, CreateQuotation 등 "데이터를 생성/수정/삭제"하는 작업을 정의합니다.
→ 만들어내는 것: "사용자가 디자인 올리기 누르면" → DB에 새 행 추가
lib/dataconnect_generated/
자동 생성 코드 (절대 수정 금지!). firebase dataconnect:sdk:generate 명령어로 schema.gql + queries.gql + mutations.gql을 분석해서 Dart 타입 코드를 자동으로 만들어줍니다.
→ 만들어내는 것: Dart에서 타입 안전하게 DB를 호출할 수 있는 함수들
🤝 협업 플로우 (Cowork v4.5) Screen
▼
lib/screens/cowork_v2/request/
셀러가 디자이너에게 협업 요청. 원하는 수량, 색상, 참고 이미지, 스타일 설명을 입력합니다. → Cowork 테이블에 status='REQUESTED'로 생성.
→ 만들어내는 것: "협업 요청하기" 화면 + 요청 데이터 DB 저장
lib/screens/cowork_v2/quotation/
디자이너가 견적서 작성. 품목명, 색상별 수량, 단가, 납기일 입력 → Quotation 테이블 생성 + Cowork status='QUOTED'. 셀러/디자이너 양쪽 수수료 미리보기 포함.
→ 만들어내는 것: 견적서 작성 폼 + 수수료 자동 계산 + PDF 생성
lib/screens/cowork_v2/payment/
토스페이먼츠 에스크로 결제. 셀러가 견적 금액을 안전결제. payment_screen.dart에 토스 SDK 연동, 협업 약관 동의 체크, B2B 거래 고지 배너.
→ 만들어내는 것: 결제 화면 → 토스 결제창 → 결제 완료 → status='PAID'
lib/screens/cowork_v2/sample/
샘플 제작 진행 추적. 디자이너가 6단계(패턴 발주 → 재단 → 봉제 → 마감 → 검수 → 완료) 순서대로 진행률을 업데이트. SampleProgress 테이블.
→ 만들어내는 것: 프로그레스 바 + 단계별 체크 UI + 완료 사진 업로드
lib/screens/cowork_v2/feedback/
셀러 피드백. 샘플 수령 후 승인(→본생산) 또는 재샘플 요청(→새 견적). 최대 2회 재샘플 가능.
→ 만들어내는 것: 승인/재샘플 선택 UI + 재샘플 사유 입력
lib/screens/cowork_v2/production/
본생산 플로우. 셀러가 생산 수량 제출 → 디자이너 본생산 견적 → 결제 → 제작 → 납품. ProductionOrder + Production 테이블.
→ 만들어내는 것: 본생산 수량 입력 → 견적 → 결제 → 진행 추적
lib/screens/cowork_v2/cs/
CS (불량 처리) 플로우. 셀러가 불량 사진 업로드 → 디자이너 건별 판정(수선가능/불가) → 수선 → 반송. CSRequest + CSItem 테이블.
→ 만들어내는 것: 불량 신고 UI + 이미지 갤러리 + 수선 결과 입력
lib/screens/cowork_v2/complete/
완료 & 리뷰. 디자이너↔셀러 상호 리뷰(별점 + 텍스트) + 정산 처리. Review + Settlement 테이블.
→ 만들어내는 것: 별점 리뷰 UI + 정산 보고서 PDF + 리오더 버튼
🎨 AI 렌더링 엔진 Core · 특허
▼
lib/services/ai_service.dart
핵심 차별점! 스케치 + 원단 → 실사 이미지 생성 (v7.5).

3-Tier 폴백 시스템:
① Gemini 3.1 Flash (Primary, $0.045/장) → ② Gemini 3 Pro (Fallback) → ③ GPT Image 1.5 (Safety Net)

스마트 재시도: 429(할당량 초과) → 즉시 다음 모델 전환, 503(서버 에러) → 지수 백오프

프롬프트 구조: 성별 + 복종 + 소분류 + 디자인 디테일 태그 + 원단 특성(투명도/질감) → 자연어 프롬프트 자동 조립
→ 만들어내는 것: 디자이너 스케치 + 원단 사진 = 실사 룩북 이미지 (특허 출원 기술)
lib/screens/upload/upload_screen.dart
디자인 업로드 화면. 스케치(앞면/뒷면) + 원단 스와치(최대 3장) + 카테고리/소분류 + 디자인 디테일 태그(8개 메인, 80+ 서브) + MOQ/가격/사이즈 입력. 3:4 비율 강제 크롭.
→ 만들어내는 것: "새 Piece 올리기" 전체 폼 + AI 렌더링 트리거

⚡ Cloud Functions (서버 로직) Infra
▼
functions/index.js
서버리스 백엔드 함수 모음. 앱에서 직접 할 수 없거나 보안이 필요한 로직을 서버에서 실행합니다:

• createKakaoCustomToken — 카카오 로그인 → Firebase Custom Token 발급
• sendVerificationSMS / verifyPhoneCode — Solapi 통해 카카오 알림톡 인증번호 발송/검증
• sendPushNotification — 협업 상태 변경 시 FCM 푸시 발송
• onChatMessageCreated — 채팅 메시지 생성 → 상대방에게 자동 푸시 (Firestore 트리거)
• verifyPaymentOwnership — 결제 소유권 검증 (보안)
• verifyBusinessNumber — 국세청 API로 사업자등록 상태 조회
• cleanupDuplicateFcmToken — FCM 중복 토큰 정리
→ 만들어내는 것: 앱이 "서버에게 부탁"하는 모든 것 — 인증, 알림, 보안 검증
